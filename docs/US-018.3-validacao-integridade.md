# US-018.3 — Validação de integridade (Sheets -> Supabase)

Data: 2026-04-30

## Objetivo

Validar se os dados sincronizados das planilhas estão chegando corretamente no Supabase antes da “virada” definitiva de leitura/gravação.

## Como rodar

1) Configure as variáveis de ambiente:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY` (recomendado) ou `SUPABASE_ANON_KEY`
- `SUPABASE_SCHEMA` (opcional, padrão: `public`)
- `GOOGLE_WEBAPP_URL` (URL do Web App do Apps Script publicado)
- `CHAVE_MESTRA` (opcional; padrão no código: `EAC-Admin-Secure-778899`)

2) Execute:

```bash
npm run validate:supabase-sync
```

O script gera arquivos em `reports/`:

- `US-018.3-sync-report-<timestamp>.json`
- `US-018.3-sync-report-<timestamp>.md`

## O que é validado (critérios de aceite)

- Contagem de registros (Sheets vs Supabase) por entidade (delta)
- Campos obrigatórios (vazios/ausentes) em Sheets e em Supabase
- Tipos mínimos:
  - datas (ISO e BR dd/mm/aaaa)
  - telefone (normalização para dígitos)
  - email (contém `@`)
  - booleanos (Sim/Não, true/false, 1/0, x)
- Duplicidades por “chave natural” (ex: telefone ou título+assunto)
- Relatório de inconsistências (MD + JSON)

## Regra de tratamento (proposta)

- Normalizar telefone para somente dígitos.
- Datas fora dos formatos aceitos -> marcar como inválidas e corrigir na origem.
- Booleanos fora do domínio aceito -> marcar como inválidos.
- Registro com campo obrigatório ausente -> quarentena (`*_invalid`) ou `status_validacao = 'INVALID'`.
- Duplicidade por chave natural -> manter 1 canônico (mais recente) e registrar conflito em log/auditoria.
