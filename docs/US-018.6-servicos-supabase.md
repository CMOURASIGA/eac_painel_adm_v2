# US-018.6 — Camada de serviço (Supabase)

Data: 2026-04-30

## Objetivo

Centralizar consultas/gravações (via backend `/api/comunicados`) em módulos de serviço, evitando `fetch` duplicado em componentes e padronizando erros/retorno vazio/loading.

## Arquivos

- Cliente padrão (tratamento de erro/JSON):
  - `services/eacApiClient.ts`
- Serviços de domínio:
  - `services/inscricoesService.ts`
  - `services/encontreirosService.ts`
  - `services/presencaService.ts`
  - `services/statusService.ts`
  - `services/auditoriaService.ts`

## Padrão de retorno

Todos os serviços retornam `EacApiResult<T>`:

- `{ success: true, data: T }`
- `{ success: false, error: string, status?: number, sample?: string }`

## Observação

Mesmo com Supabase como fonte oficial de leitura, algumas ações ainda podem ser legadas (Apps Script) enquanto a fase de gravação/execução não for migrada. Os serviços encapsulam isso por `action`.
