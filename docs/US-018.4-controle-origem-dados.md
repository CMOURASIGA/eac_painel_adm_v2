# US-018.4 — Controle de origem dos dados (PLANILHA vs SISTEMA)

Data: 2026-04-30

## Objetivo

Durante a transição (Sheets -> Supabase), precisamos auditar de onde veio cada registro e impedir que a sincronização sobrescreva dados criados diretamente pelo sistema sem regra definida.

## Campos (propostos)

- `origem_dado` (**obrigatório**): `PLANILHA` ou `SISTEMA`
- `data_importacao` (timestamptz): quando o registro foi importado da planilha
- `id_origem_planilha` (text): identificador do “row”/aba/origem na planilha
- `ultima_sincronizacao` (timestamptz): quando o registro foi atualizado pela última sincronização
- `criado_via_sistema` (boolean): redundante/derivado de `origem_dado`, mas útil para filtros rápidos

## Regras (aceite)

1) Todo registro importado deve ter `origem_dado = 'PLANILHA'`
2) Todo registro criado pelo sistema deve ter `origem_dado = 'SISTEMA'`
3) O sistema deve permitir auditoria visual da origem (UI)
4) A sincronização **não pode** sobrescrever registros `SISTEMA` sem regra explícita
5) Deve existir data/hora da última importação/atualização

## Implementação (aplicação)

O painel já exibe os metadados quando presentes no Supabase (em modais/drawers):

- `components/MembersPage.tsx` (aba “Auditoria”)
- `components/InscricoesPrioritariasPage.tsx` (seção auditoria no drawer)
- `components/EncontreiroPage.tsx` (modal de detalhes)
- `components/CalendarPage.tsx` (modal de evento)
- `components/ComunicadosPage.tsx` (modal de comunicado)

## Implementação (banco de dados)

O SQL recomendado está em `docs/US-018.4-controle-origem-dados.sql`.

Observações:

- Para inserts feitos pela aplicação, recomenda-se `origem_dado = 'SISTEMA'` e `criado_via_sistema = true`.
- Para importação/sync, o processo deve setar explicitamente `origem_dado = 'PLANILHA'`, preencher `data_importacao`, `id_origem_planilha` e atualizar `ultima_sincronizacao`.
- Para proteger dados, o upsert do import deve atualizar somente registros cuja `origem_dado = 'PLANILHA'` (não tocar nos `SISTEMA`).
